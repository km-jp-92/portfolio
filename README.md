■サービス概要  
リアル会議効率化ツール。PDFページ同期、チャット、個人メモ機能から構成。アカウント登録不要。

■ このサービスへの思い・作りたい理由  
現職において、外部のペーパレス会議システムを導入しており、私はその運用を担当していたことがありました。便利なシステムであり、自分でも作ってみたいと考えました。また、法人が利用するシステムでしたので利用には詳細な設定などが必要でした。そのため、より機能を簡便にして個人も含めて使ってもらえるサービスが作れないかと考えました。

■ ユーザー層について  
リアルの会議・イベント・勉強会などの主催者
> リアルの会議、イベント、勉強会、打ち合わせなどでペーパレスで資料を共有したいニーズが一定数存在すると考えました。

■サービスの利用イメージ
- リアル会議・イベント・セミナー
> 会場で配布資料を紙で渡さず、URLやQRで閲覧させたい  
> 「参加者が自由に端末で見れる＋発表に同期もできる」が便利

- 教育・講習系
> 授業・研修で、受講者が自分の端末で資料を見ながら進行に合わせたい場合

■ ユーザーの獲得について
- まずは自分が参加する身近な人（家族・友人）やRUNTEQの他の受講生の方に直接ヒアリングしてフィードバックを得る（GoogleフォームやNotionで簡単なアンケートを用意）。
- その後、必要最低限の改善を行った後、問題がなければSNSで初期ユーザー獲得を目指す。

■ サービスの差別化ポイント・推しポイント
- 利用が想定されるのは特殊な場面であり、同じようなサービスは少ない。
- 既存のペーパーレス会議システムもフォーマルな会議用途がメインとなっているため、差別化が一定図れる。
- Zoomのようなオンライン会議ツールとは用途が異なる。

■ 機能  
- 認証（メールアドレス入力→メール送付→メール記載のURLからパスワード設定）
- パスワード再設定
- 資料のアップロード、削除
- 発表者が資料をページ送りすると、聴講者がリアルタイム同期する機能
- チャット
- メモ、AI整形

■ 機能の実装方針  
フレームワーク・言語
- Ruby 3.3.6、Ruby on Rails 7.2.3

フロントエンド
- Tailwind CSS
- React 19.2.0

バックエンド
- Rails ActionCable（資料ページのリアルタイム同期、チャット）

データベース
- PostgreSQL（本番環境: Render / 開発環境: ローカル PostgreSQL）

ストレージ
- Amazon S3（ファイルアップロード・資料保存）

メール送信
- Resend

デプロイ
- Render（Railsアプリ + PostgreSQL + Redis）

認証 / セキュリティ
- パスワード生成ページURL（トークン）の有効期限設定
- パスワードハッシュ化（has_secure_password、BCrypt）
- 資料格納・閲覧ページのパスワード認証、有効期限設定
- PDFのURL（Amazon S3）への直接アクセス防止
- レート制限（Rack::Attack）

■ 設計について  
1. ユーザーがトップページでメールアドレスを入力
2. サーバーが「パスワード生成用URL」、「資料アップロードURL」、「資料閲覧URL」の３つのURLをそれぞれ生成して当該ユーザーのメールアドレスに送る
3. 該当ユーザーは届いたメールに記載されたURLにアクセスし自分が決めたパスワードを入力すれば入室可能
4. 該当ユーザーが「資料アップロードURL」に資料をアップロードする（もしくは適宜参加者に「資料アップロードURL」とパスワードを共有し、資料アップロードをしてもらう）
5. 該当ユーザーが参加者に「資料閲覧URL」とパスワードを共有。参加者がそのURLにアクセスすると、サーバーが資料アップロードURLにアップロード済みのPDFを参照して表示。「発表者になる」ボタンを押すと、発表者のPDFページが聴講者に同期される

■ PDFのリアルタイム同期の実装について  
- 「発表者になる」ボタンを押す
- フロント側で、ボタンが押されたかどうかを使ってフラグを切り替える。押されていれば自分のページ番号がActionCableを通じて「聴講者になる」ボタンを押した全ブラウザに送信される。
